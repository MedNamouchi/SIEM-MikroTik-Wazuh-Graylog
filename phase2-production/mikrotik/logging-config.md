# MikroTik Logging Configuration — Phase 2 (Production)

Configuration of remote logging for physical CCR routers.

> **Key difference from Phase 1:** Routers now use their real hostname
> as prefix (via `$routername`) instead of fixed numeric IDs like `MKT[001]`.
> This means the prefix is set dynamically and reflects the actual router identity.

---

## Prerequisites

- RouterOS **7.23+**
- Graylog TLS certificate (`graylog.crt`) transferred to the router
- Graylog CEF TCP input running on port **6514**

---

## Step 1 — Transfer the Certificate

```bash
scp /etc/graylog/server/certs/graylog.crt admin@ROUTER_IP:graylog.crt
```

---

## Step 2 — Import the Certificate on MikroTik

```routeros
/certificate import file-name=graylog.crt passphrase=""
```

Verify:

```routeros
/certificate print detail
```

Expected output must show:
```
key-usage: digital-signature, key-encipherment, tls-server
trusted: yes
```

> ⚠️ If `tls-server` is missing, regenerate the certificate with `extendedKeyUsage=serverAuth`.

---

## Step 3 — Create the Logging Action

> ⚠️ `remote-protocol=tls` MUST be set in the `add` command.
> RouterOS returns a syntax error if you try to set it afterwards with `set`.

```routeros
/system logging action add \
  name=graylogcef \
  target=remote \
  remote=GRAYLOG_SERVER_IP \
  remote-port=6514 \
  remote-log-format=cef \
  remote-protocol=tls
```

Verify:

```routeros
/system logging action print detail
# Expected: remote-protocol=tls
```

---

## Step 4 — Configure Logging Topics

Unlike Phase 1 where each router had a hardcoded prefix (`MKT[001]`...),
Phase 2 uses `$routername` to dynamically read the router's hostname.
This way the same script works on every router without modification.

```routeros
:local routername [/system identity get name]

/system logging add topics=firewall prefix="$routername[FW]"   action=graylogcef
/system logging add topics=system   prefix="$routername[SYS]"  action=graylogcef
/system logging add topics=info     prefix="$routername[INFO]" action=graylogcef
/system logging add topics=warning  prefix="$routername[WARN]" action=graylogcef
```

> Note: The `warning` topic is new in Phase 2 — it captures CPU and RAM
> alerts generated by the resource monitor script.

---

## Verification

```bash
sudo ss -tulnp | grep 6514
# Expected: tcp LISTEN 0 4096 *:6514 *:* users:(("java",...))
```

Then go to **Graylog → Search** and filter by `device_vendor:MikroTik`.
Logs should appear within seconds of any router activity.
